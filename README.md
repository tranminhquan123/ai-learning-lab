# AI Learning Lab

This repository is my personal learning system for AI Engineering, AI Application Development, finance/investing education, Chinese for work, and DaVinci Resolve.

The purpose is not only to store notes. The purpose is to give Claude enough structure, memory, prompts, skills, and resources to support long-term learning without restarting from zero every time.

My basic Python roadmap is managed separately. This repo starts from the stage after Python basics and focuses on AI application development and supporting skills.

## Main Goals

- Build foundations in AI, Machine Learning, and LLMs.
- Learn how to build AI applications with Python.
- Set up Claude Code, MCP, memory, and agent workflows.
- Learn finance and investing from reliable sources.
- Learn Chinese for work and communication.
- Learn DaVinci Resolve through practical editing workflows.

## Repository Structure

- `CLAUDE.md`: core instructions for Claude and Claude Code.
- `memory/`: learning state, mistakes, decisions, project progress, weekly reviews.
- `roadmap/`: long-term learning paths outside the basic Python roadmap.
- `notes/`: subject notes and explanations.
- `projects/`: practical projects and experiments.
- `prompts/`: reusable prompts for Claude.
- `skills/`: custom Claude Skills.
- `resources/`: trusted docs, books, papers, and GitHub repositories.
- `configs/`: Claude Code, MCP, memory, and environment configuration notes.

## How Claude Should Use This Repo

When working inside this repo, Claude should read these files first:

1. `CLAUDE.md`
2. `memory/learning-state.md`
3. `memory/decisions.md`
4. `memory/projects-log.md`
5. Relevant files in `roadmap/`

Then Claude should continue from the current learning state instead of starting over.

Do not fork every useful repository.

- Tool or package repositories should be installed with package managers or plugin commands.
- Course and documentation repositories should be cloned locally for reading and practice.
- Personal learning work should be saved in this repository.
- Fork only when modifying a repository or contributing pull requests.
